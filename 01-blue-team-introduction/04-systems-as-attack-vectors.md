# Systems as Attack Vectors

## Room Link
https://tryhackme.com/room/systemsattackvectors

---

## Objective
Understand how attackers exploit vulnerabilities in systems, software, and configurations to gain unauthorized access.

---

## Key Concepts

### What Are System Attack Vectors?
System attack vectors target **technical weaknesses in systems**, such as servers, applications, or networks.

✅ Key Idea:  
Attackers do not need user interaction; if a system is insecure, they can exploit it directly.

---

### Impact of System Breaches

The impact of a system compromise depends on its value:

- Personal device → limited damage  
- Admin workstation → access to internal systems  
- Mail server → compromise of all users  
- Critical infrastructure → large-scale disruption  

✅ Key Idea:  
A single system breach can have **major consequences depending on its role**

---

### Common Ways Systems Are Compromised

#### 1. Human-Led Actions
Even system attacks often begin with user mistakes:
- Weak or reused passwords  
- Downloading malicious software  
- Inserting infected USB devices  

✅ Fact:  
Most breaches involve compromised credentials.

---

#### 2. Vulnerabilities
A **vulnerability** is a flaw in software that attackers can exploit.

- Public vulnerabilities are assigned **CVE IDs**  
- Unknown vulnerabilities are called **zero-days**  
- Attackers often exploit systems before patches are applied  

✅ Key Idea:  
There is a race between attackers exploiting vulnerabilities and defenders patching them

---

#### 3. Supply Chain Attacks
Attackers compromise trusted software or libraries to distribute malware.

Examples:
- Compromised app updates  
- Malicious dependencies  

✅ Key Idea:  
Even trusted software can become a threat

---

#### 4. Misconfigurations
Security issues caused by incorrect system setup (not software bugs)

Examples:
- Weak passwords  
- Exposed databases  
- Open network access  

✅ Key Idea:  
Misconfigurations are one of the most common and preventable risks

---

## Responding to System Threats

### Patch Management
- Apply updates to fix vulnerabilities  
- Critical for protecting against known exploits  

---

### Monitoring & Detection
- Watch for suspicious activity  
- Detect exploitation attempts early  

---

### Securing Configurations
- Enforce strong passwords  
- Restrict access to trusted users/IPs  
- Follow security best practices  

---

### Proactive Security Measures
- **Vulnerability Scanning** → identify weaknesses  
- **Penetration Testing** → simulate real attacks  
- **Configuration Audits** → ensure proper setup  

---

## Takeaways

- System attacks can occur without user involvement  
- Vulnerabilities and misconfigurations are major risks  
- A single compromised system can impact entire organizations  
- Security depends heavily on patching and proper configuration
